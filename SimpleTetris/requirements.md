## 功能需求
* 使用 Observation framework 管理遊戲狀態。(@Observable)
* 使用 environment 存取 Observable object。
* 使用 sheet 或 fullScreenCover 切換頁面。
* Replay 功能，遊戲結束時可再玩一次。
* 使用 Timer、TimelineView 或 Task.sleep，比方實現計時或倒數功能。
* 加入音樂或音效。

---

## 加分功能
* 動畫。
* 遊戲的設定頁面。
* 遊戲的教學頁面。
* 遊戲的成績記錄，比方分數的排行榜。

## 以俄羅斯方塊為主題的遊戲基礎需求
* 需要一個 Timer 去更新整個畫面
* 整個遊戲應用程式點進去後會去視情況自動打開教學介面(TutorialView)
    * 利用 SwiftData 存取變數 ShowTutorial(boolean)，當它為 true 時會顯示教學畫面，反之則否
    * 初始設為 true，當使用者點擊教學畫面的 "關閉按鈕" 後，設為 false 
    * 先把教學介面做好再來製作這個變數，利用這個變數去顯示教學介面
* User Interface 講究簡潔
    * 使用單頁面形式顯示兩個頁面，分別為：
        1. 遊戲頁面(GameView)
            * 左上角顯示分數
            * 右上角顯示設定按鈕，點擊後導至設定頁面(SettingView) (在那可以再透過點擊左上角的返回按鈕返回遊戲畫面(GameView))
            * 方塊會自動向下移動
            * "不"顯示上下左右控制按鈕
                * 但是 user 可以透過向左滑跟向右滑來滑動螢幕，這會導致正在掉落中的方塊向左移動一格或是向右移動一格
                * 同時，前面有提到方塊會向下移動，如果 user 長按螢幕，那麼方塊會加速向下移動，加速倍率預設為 x2
            * 禁止任何方塊的向上移動（可以單純不要去撰寫任何向上移動的程式碼就好）
        2. 設定頁面(SettingView)
            * 可以查看遊戲紀錄，還有過去遊玩的成績，以及最好的成績等等
            * 可以調整 user 長按螢幕時，向下移動的倍率，最低 x1.0(也就是長按不會在造成向下移動速率有任何增加)，最高 x5.0
            * 支援設定遊玩時的背景圖片，讓 user 可以上傳圖片並保留在 SwiftData 內
            * 如果有任何需要調整的也可以加入到設定頁面
    * 不考慮教學頁面的話，點開 App 後，User 會在 GameView 內，然後點擊中央開始按鈕後開始遊戲
    * 在 GameView 使用圖片當作背景
